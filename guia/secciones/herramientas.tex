\section{Wireless Tools}
Wireless Tools es un paquete de herramientas de verificación y configuración de redes Wifi, sus principales aplicaciones son \texttt{iwconfig, iwlist, iwspy, iwpriv, iwgetid, ifrename e iwevent}. La herramienta utilizada en el desarrollo del proyecto ha sido iwlist.\\
 Iwlist se utiliza para mostrar información adicional de las interfaces de redes inalámbricas detectadas. El principal argumento se utiliza para seleccionar una categoría de información, iwlist muestra en forma detallada toda la información relacionada con esta categoría.\\

\textbf{Parámetro scan}
Devuelve la lista de puntos de acceso y celdas Ad-Hoc dentro de rango y, opcionalmente, información sobre ellos (ESSID, calidad, frecuencia, modo \ldots). El tipo de información devuelta depende de lo que admite la tarjeta, generalmente muestra la dirección MAC del punto de acceso, su nombre de red o ESSID, canal en que realiza la transmisión, ratio de transferencia y la encriptación de la red.

\section{GnuPlot}
\subsection{Generación de gráficos}
%%%%%%%%%%%%%%%%%%%%%%%%%
Gnuplot es un programa de línea de comandos que puede generar gráficos de funciones en dos y tres dimensiones. Se utiliza con frecuencia para los gráficos con calidad de publicación, así como los propósitos educativos. Gnuplot puede producir una salida directamente en la pantalla, o en muchos formatos de archivos gráficos, incluyendo Portable Network Graphics (PNG), PostScript encapsulado (EPS), Scalable Vector Graphics (SVG), JPEG y muchos otros. El programa puede ser utilizado tanto de forma interactiva y en modo batch utilizando scripts.\\
%%%%%%%%%%%%%%%%%%%%%%%%%
Fue creado originalmente para permitir a los científicos y los estudiantes a visualizar funciones matemáticas y datos de manera interactiva, pero ha crecido para apoyar muchas aplicaciones no interactivas, tales como secuencias de comandos web. También se utiliza como un motor de trazado por las aplicaciones de terceros como octava. Gnuplot ha contado con el apoyo y bajo desarrollo activo desde 1986.\\

Gnuplot es compatible con muchos tipos diferentes de salida: pantalla de los terminales interactivos (con tecla de acceso directo y la entrada del mouse), salida directa a plotters o impresoras modernas, y la salida a muchos formatos de archivo (eps, fig, jpeg, LaTeX, metafont, PBM, pdf, png , PostScript, SVG, \ldots). \\

\section{Qt}
\subsection{Descripción de Qt}
Las librerias gráficas de Qt fueron desarrolladas para disponer de un GUI para C++ multiplataforma por la compañia Trolltech. Qt cambió a licencia GPL en el año 2000, siendo gratuita para el software libre y de pago para el privativo, a 
partir de la compra de Trolltech por parte de Nokia en 2008.\\

%\figura{0.4}{imgs/capturas/qt-logo.jpg}{Logotipo de Qt}{}{}

\figuraSinMarco{0.4}{imgs/capturas/qt-logo.jpg}{Logotipo de Qt}{Logoqt}{h}

Qt fue utilizada para el desarrollo de KDE entre 1996 y 1998, y éste se convirtió en un escritorio muy popular entre la comunidad GNU/Linux. Esto provocó la aparición del escritorio GNOME con GTK+ debido a que la licencia de Qt se basaba en software propietario. En 1998 los desarrolladores de KDE y Trolltech establecieron la KDE Free Qt Foundation, la cual permitía liberar la biblioteca Qt bajo una licencia tipo BSD10.En el año 2000, Trolltech pasó a ofrecer la biblioteca Qt bajo GPL para la versión Linux, en Mac OS X se publico en 2003 y en MS Windows en 2005. \\

Actualmente, Qt cuenta con tres tipos de licencia; 
\begin{itemize}
\item GPL para el desarrollo de código abierto y software libre
\item QPL de pago para el desarrollo de aplicaciones comerciales
\item LGPL una licencia gratuita para aplicaciones comerciales
\end{itemize}

Qt está disponible en Ms Windows y en los sistemas tipo Unix como Linux, BSD, Unix, Mac OS X, también en sistemas embebidos (PDA, Smartphones\ldots) como Linux embedded, Symbian y Ms windows CE. Actualmente, Nokia ha anunciado que no utilizará Qt en sus nuevos terminales con WP7.\\

\subsection{Herramientas Qt utilizadas}

Para el desarrollo del proyecto, se ha utilizado el IDE Qt Creator, que incluye compilador, herramientas para el diseño de la GUI, debugger (GDB)\ldots También dispone de ayudas como indentación automática, resaltado de código, integración con servicios de control de versiones Git\footnote{http://git-scm.com}, Subversion\footnote{http://subversion.tigris.org}, Perforce\footnote{http://www.perforce.com}, CVS\footnote{http://www.cvshome.org} y Mercurial\footnote{http://mercurial.selenic.com}.\\

Qt Creator es una herramienta para el diseño y la creación de interfaces gráficas de usuario (GUI) de widgets Qt. Qt puede componer y personalizar widgets o cuadros de diálogo, utilizando diferentes estilos y resoluciones.
Los widgets y formularios creados con Qt Creator se integran sin problemas con el código de programación, mediante el mecanismo de señales y ranuras (signal y slot, respectivamente), que permite asignar fácilmente el comportamiento de los elementos gráficos. Todas las propiedades y comportamientos establecidos en Qt Creator se pueden cambiar dinámicamente en el código. \\

\figuraSinMarco{0.9}{imgs/capturas/qtcreator.png}{Pantalla principal de QtCreator}{qtcreator}{h}

Qt Creator permite crear y ejecutar aplicaciones. Se entiende el código como código, no sólo como texto sin formato. Esto le permite:
\begin{itemize}
\item Escribir código bien formateado e indentado.
\item Anticipar lo que va a escribir y completar el código.
\item Visualización en línea de error y mensajes de advertencia.
\item Le permiten navegar semánticamente a las definiciones y declaraciones de clases, funciones y símbolos.
\item El desarrollador recibe ayuda sensible al contexto de las clases, funciones y símbolos.
\item Mostrar las ubicaciones en el código donde se declara una función o una llamada.\\
\end{itemize}


Qt proporciona un excelente soporte para la traducción de las aplicaciones a las lenguas locales. La herramienta de Qt de traducción necesaria para ello es Qt Linguist, que se combina con \textit{lupdate} y \textit{lrelease}.  La herramienta \textit{lupdate} se utiliza para sincronizar el código fuente y traducciones. La herramienta \textit{lrelease} se utiliza para crear archivos en tiempo de ejecución de traducción para el uso de la aplicación en libertad.\\

La mayor parte del texto que debe traducirse en un programa de aplicación consta de cualquiera de las palabras sueltas o frases cortas. Estos suelen aparecer como títulos de las ventanas, elementos de menú, texto de ayuda emergente (ventana de ayuda), y las etiquetas a los botones, casillas de verificación y botones de radio.\\

\figura{0.4}{imgs/capturas/qt-linguist.png}{Logotipo de Qt-Linguist}{Logoqtling}{h}


Las frases se introducen en el código fuente por el programador con una sintaxis sencilla, pero especial para identificar las frases que requieren traducción. Las herramientas de Qt proporcionan información de contexto para cada una de las frases para ayudar al traductor, y el programador es capaz de agregar información de contexto adicional a las frases cuando sea necesario. El encargado de la liberación genera un conjunto de archivos de traducción que son producidos a partir de los archivos de origen de pasar estos al traductor.\\

El traductor abre la traducción en los archivos utilizando Qt Linguist, entra en sus traducciones y guarda los resultados de nuevo en los archivos de traducción, que pasan de nuevo al encargado de la liberación. El encargado de la liberación a continuación, genera rápidamente versiones compactas de estas traducciones archivos listos para su uso por la aplicación. Las herramientas están diseñadas para ser utilizado en ciclos repetidos como las aplicaciones cambian y evolucionan, la preservación de las traducciones existentes y lo que es fácil de identificar que las nuevas traducciones se requieren.\\
