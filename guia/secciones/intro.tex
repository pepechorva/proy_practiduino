\section{Descripción}
Esta herramienta proporciona información acerca de los puntos de acceso colindantes para el análisis de intensidad de señales y el uso de canales de transmisión. También proporciona estos informes en soporte portable (PDF) o html. Apoyándose en herramientas libres tales como iwlist, Gnuplot, \LaTeX y Qt se generan las estadísticas de los puntos de acceso.\\

\section{Motivación}
Durante la instalación de redes inalámbricas, es importante verificar que la señal llega correctamente hasta el lugar que nos interese, y que las interferencias causadas por las redes colindantes sean las mínimas posibles, para que la transmisión sea lo más eficiente posible.\\

Éste escáner de redes inalámbricas permite solucionar de problemas de una red inalámbrica existente o analizar cual o cuales llegan con más potencia a nuestro receptor. También permite saber cuántas redes hay en el entorno y la potencia de sus señales, para verificar la cobertura wifi de un determinado espacio, la posibilidad del roaming, la presencia de interferencias de radiofrecuencia o "puntos muertos", y la ubicación óptima de los puntos de acceso y las antenas.\\

Esta herramienta de diagnóstico correspondería a la categoría de escáneres WiFi. El más conocido de esta categoría es NetStumbler. Una herramienta de informes de cada red inalámbrica que detecta, junto con el canal utilizado por el punto de acceso (AP), la potencia de la señal recibida\ldots \\

La forma en que esto funciona es que aproximadamente cada 100 ms un punto de acceso envía una señal y la tarjeta inalámbrica 802.11, también conocida tarjeta wifi o ``wifi'' la detecta, y añade el SSID a su lista de redes inalámbricas conocidas. Además, es posible obtener los informes para cada AP individualmente, es decir, los que recibe en su ubicación actual.\\


Este programa ha sido desarrollado con las siguientes herramientas libres:\\

\begin{itemize}
\item Iwtools
\item GnuPlot
\item Plataforma de desarrollo Qt
\item \LaTeX
\end{itemize}

Las principales ventajas de utilizar herramientas dentro del programa son más facilidad de mantener el código actualizado,   y reutilización de recursos, ya que es posible que algunas de estas herramientas ya se encuentren instaladas en el sistema operativo.\\


\section{Objetivo}
\newcommand{\wifi}{WifiScan}
La herramienta de desarrollo (en adelante, \wifi) está basada en el programa para Microsoft Windows NetSurveyor\footnote{http://www.performancewifi.net/performance-wifi/products/netsurveyor-network-discovery.htm}. Este proporciona información gráfica de las distintas redes detectadas y permite una interpretación casi intuitiva. \\

El objetivo principal de éste proyecto es hacer una interfaz gráfica para combinar herramientas libres de análisis de redes inalámbricas con otras de muestreo estadístico, utilizando una plataforma de desarrollo multiplataforma para tener la opción de hacer versiones para otros sistemas operativos.\\

La creación de \guis \ tiene una gran relevancia a la hora de interactuar con un programa, permitiendo incluso, un mayor entendimiento de los datos a analizar, como en éste caso. En el desarrollo de \guis \ es importante tener en cuenta la funcionalidad del programa frente a querer mostrar todos los datos sin aparente criterio o de tener que recurrir a comandos de consola para seleccionar los datos que nos interesen en un momento determinado.\\
